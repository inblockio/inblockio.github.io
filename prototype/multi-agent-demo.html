Multi-Agent Business Transaction

Theodora orders a product via her agent Symphonie. Best-Cookies Inc. responds through their agent Brandon. Every step produces a signed, hash-chained artifact that either party can independently verify.

Transaction Flow (from Aqua Protocol whitepaper)

  1. User Intent — Theodora signs her purchase intent through Symphonie
  2. Offer — Brandon retrieves offerings from Best-Cookies API, returns signed artifact
  3. Order — Symphonie creates order referencing the offer by hash-link
  4. Confirmation — Brandon processes against API, links back to order
  5. Invoice — Final artifact, hash-chained to confirmation

Each artifact carries three signature layers: User (P-256), Agent (Ed25519), Workflow (Ed25519).
Hash-chain links (golden curves) create a tamper-evident transaction record.

Data Notarization Pipeline

The underlying mechanism: data enters, gets form templates, eSignatures, and timestamps applied, sealed in a certified box and shipped. On receipt, the box is unboxed, inspected, and verified — or rejected if corrupt.

Every 3rd cycle shows a corrupt/failed verification (red cross).
